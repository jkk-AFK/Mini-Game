创建一个基于 Web 的小游戏平台，包含多个经典小游戏（如马里奥、俄罗斯方块、贪吃蛇、井字棋等）。平台支持用户注册、登录、选择游戏、进行游戏并与其他玩家互动。需求如下：

前端需求：

使用 HTML、CSS 和 JavaScript（React.js 或 Vue.js）。

首页展示所有可用游戏（如马里奥、俄罗斯方块等），支持游戏控制（暂停、开始等）。

登录后，用户查看游戏成绩和平台排名。

游戏页面响应式设计，适配桌面、平板、手机等设备。

支持电子邮件、Google、Facebook 等社交账户注册和登录。

游戏页面有简洁的控制界面，包括暂停、开始、重玩、设置等按钮。

后端需求：

使用 Node.js 和 Express.js 创建 RESTful API。

提供用户身份验证功能（JWT 或 OAuth2）。

存储用户信息（用户名、邮箱、密码等）和游戏历史记录。

存储游戏成绩、排行榜数据，并支持 API 查询和更新。

支持 WebSocket 实时通信，支持多人对战。

数据库需求：

使用 MongoDB 或 PostgreSQL 存储用户数据和游戏成绩。

设计用户数据表，包含用户名、密码、邮箱、注册日期等。

设计游戏成绩表，记录每个游戏的得分、排名、游戏类型等信息。

支持根据游戏类型和得分进行查询和更新。

游戏模块需求：

游戏包括经典的 Web 版游戏：马里奥、俄罗斯方块、贪吃蛇、井字棋。

游戏支持单人模式和多人模式：

马里奥： 横版过关游戏，支持角色控制、敌人、障碍物、关卡设计等。

俄罗斯方块： 方块旋转、移动、消除行等。

贪吃蛇： 用户控制蛇的移动，吃食物增长，避免碰撞。

井字棋： 支持单人和多人对战，简单的游戏规则和胜负判定。

每个游戏提供开始、暂停、重新开始等控制。

游戏结果实时更新并展示用户得分和排名。

游戏界面适应不同设备，支持键盘、鼠标、触摸屏操作。

其他需求：

提供管理员后台，管理员可管理用户、查看统计数据、删除不良内容等。

游戏界面简洁易操作，避免复杂选项。

优化加载速度，保证低性能设备流畅运行。

平台安全，防止恶意攻击、作弊行为和数据泄露。

游戏页面支持多语言和本地化。

编写以下代码框架：

前端： 使用 React.js 或 Vue.js 构建应用架构，包括用户注册、登录、游戏选择和游戏页面。

后端： 使用 Node.js 和 Express.js 实现 API，支持用户认证、成绩管理、游戏对战等。

数据库设计： 设计 MongoDB 或 PostgreSQL 用户数据表、成绩表等，连接前后端。

游戏模块： 实现经典游戏（如俄罗斯方块、马里奥、贪吃蛇），并展示如何集成到平台中。

注意：

游戏的物理引擎和动画效果接近经典游戏，尽量还原体验。

游戏逻辑考虑不同屏幕尺寸，确保流畅体验。

多人对战支持实时同步和低延迟，确保操作实时性。
